## 功能特性

### 核心功能
- 多任务队列管理（顺序执行）
- 每个任务支持：名称、预估时长、休息次数、休息时长
- 工作/休息周期自动切换
- Windows 系统通知栏 Toast 消息提醒

### 任务管理
- 添加/删除任务
- 批量导入任务（支持 `- [ ]` 前缀格式）
- 替换所有任务 / 新增到现有任务
- 清空所有任务
- **拖拽排序**：拖拽调整任务顺序（v1.3 新增）

### 控制功能
- **开始/暂停/继续**：统一的三阶段暂停/继续逻辑（v1.6 重构）
- **跳过当前任务**：静默执行，不触发休息
- **提前完成任务**：跳过所有剩余工作段和休息段，直接进入任务间休整（v1.3 新增）
- **提前完成本工作段**：结束当前段，进入下一预定段（v1.7 新增）
- **跳到选定任务**：将目标任务提升到第一位并立即执行（v1.3 新增）
- **修改任务**：修改当前任务重置执行，修改后续任务直接应用（v1.7 重构）

### 暂停/继续功能（v1.6 重构）
- **三阶段支持**：工作中、休息中、任务间休息均可暂停/继续
- **状态记录**：暂停时记录当前阶段和剩余秒数
- **继续恢复**：从暂停时记录的阶段和剩余时间恢复
- **修改后暂停**：点击"修改"时先暂停计时，编辑后由用户点击"继续"恢复

### 任务结束休息功能（v1.5 新增）
- **自动休息**：任务完成后自动进入休息整顿（默认5分钟，可配置）
- **休息期间修改**：可修改当前任务的预估时长并重新执行
- **跳过不触发**：跳过任务不会触发休息
- **可配置时间**：标题栏提供输入框，可自定义任务间休整时间

### 日志统计功能
- **自动记录**：任务完成/跳过时自动保存到本地 JSON 文件
- **存储方式**：按日期分文件存储（`logs/YYYY-MM-DD.json`）
- **日历视图**：月历样式，支持月份切换
- **颜色标记**：
  - 🟩 深绿：全部完成
  - 🟨 黄色：有未完成任务
  - ⬜ 灰色：无任务
- **月度统计**：任务数、完成率、专注时长、休息时长等
- **日志导出**：支持导出为 CSV 格式
- **跳转终止记录**：支持 `jump_terminated` 完成类型（v1.3 新增）

### 时长统计显示（v1.5 新增）
- **任务总时长**：所有任务的预估时长 + 任务内休息时长
- **任务间休整总时间**：(未完成任务数 - 1) × 用户设置的休整时间
- **总安排时长**：任务总时长 + 任务间休整总时间（加粗显示）

### 拖拽排序规则（v1.3 新增，v1.8 完善）
- **未计时状态**：
  - 所有未完成任务可自由拖拽排序
  - 不能拖拽已完成任务之前的任务（保持已完成任务在队列前面）
- **计时中状态**：
  - 正在执行的任务（第一行）位置锁定，不可拖拽
  - 其他任务只能拖到第二行及之后的位置
  - 不能拖拽已完成任务之前的任务
- **视觉反馈**：
  - 拖拽时源任务高亮绿色
  - 目标位置高亮黄色
- **数据同步**：
  - 拖拽前自动同步输入框值到任务属性，避免数值丢失
  - 拖拽后自动更新 `current_task_index`，确保执行顺序正确

### 跳转任务规则（v1.3 新增）
- 目标任务自动提升到第一位并开始执行
- 当前任务扣除已消耗时间后移到第二位
- 自动记录当前任务的实际专注时长和已用休息次数

### 窗口调整功能（v1.4 新增）
- **可调整大小**：窗口支持自由拖拽调整宽高
- **最小尺寸**：920 × 500 像素
- **自适应布局**：内部控件随窗口大小自动适配
- **边框光标修复**：使用 Windows API 修复 Windows 11 边框调整光标不显示的问题

### 视觉设计
- 黑/白/灰主色调
- 绿色时间数字（#22C55E）
- 当前任务：深绿色高亮框
- 已完成/跳过任务：浅绿色背景
- 表头固定吸顶
- 自定义窗口图标（时钟样式）
- 拖拽手柄：⋮⋮ 符号

## 使用说明

### 源码运行
```bash
python timer_app.py
```

### EXE 运行
双击 `dist\多任务队列计时闹钟.exe`

### 打包命令
```bash
pyinstaller --clean "多任务队列计时闹钟.spec"
```

### 日志存储
- 日志自动保存在 EXE 所在目录的 `logs/` 文件夹
- 每天一个 JSON 文件，按日期命名
- 批量替换任务会清除当天日志

## 任务完成类型

| 类型 | 说明 |
|------|------|
| `normal_complete` | 正常完成（所有工作段执行完毕） |
| `early_complete` | 提前完成（用户主动点击提前完成按钮） |
| `skipped` | 跳过（用户主动跳过任务） |
| `jump_terminated` | 跳转终止（因跳转到其他任务而被终止） |

## 按钮对比

| 按钮 | 作用对象 | 行为 |
|------|---------|------|
| **提前完成任务** | 整个任务 | 跳过所有剩余工作段和休息段，直接进入任务间休整段 |
| **提前完成本工作段** | 当前段 | 结束当前段，进入下一预定段 |
